📱 Why add mobile?
⚡
Time-sensitive alerts arrive instantly. When a $1,200 item is marked delivered, the buyer needs to know right now — not the next time they check email.
🔒
Numbers are private. Each party's phone is used only for their own alerts. Neither party sees the other's number.
💰
Cost is negligible. ~$0.08/SMS via Twilio NZ. A $40 SafeSend fee covers ~500 SMS messages.
📉
Reduces disputes. The most common dispute trigger is "I didn't know it was delivered." SMS eliminates that excuse.
🚫 What NOT to do
Don't show buyer's phone to seller or vice versa — even with consent. Once they can call each other directly, they can bypass SafeSend and do the deal without paying the fee. Keep communication in-app and use phone numbers only for outbound notifications from SafeSend itself.

🔒 Messages are stored by SafeSend and may be reviewed if a dispute is raised.
Why in-app messaging beats sharing phone numbers
✓
Dispute evidence. Every message is timestamped and stored. If a buyer claims they never received a shipping notification, the thread proves otherwise.
✓
Deal stays on platform. No risk of parties agreeing to bypass escrow once they're messaging outside SafeSend.
✓
Privacy protected. Neither party's contact details are exposed. Admin can read the thread if a dispute is raised.
✓
System messages inline. State transitions (funded, shipped, delivered) appear in the thread automatically — everyone sees the same timeline.
What to build — v1 vs v2
BUILD NOW
Simple async thread — messages stored in DB, email notification to other party on each new message with a "View in SafeSend" link. No WebSockets needed. Both parties check the deal page to see replies. This works fine for deals that take 2–5 days.

V2 LATER
Real-time chat with WebSocket push notifications. Only needed if you have high-frequency traders doing multiple deals per day.

📧 Resend (Email)

Already installed. Just add more template functions to email.ts
// email.ts — add alongside sendMagicLinkEmail export async function sendDealFundedEmail( sellerEmail: string, deal: Deal ): Promise<void> { await resend.emails.send({ from: "SafeSend <noreply@safesend.nz>", to: sellerEmail, subject: `Payment received — ship "${deal.title}" now`, html: fundedEmailHtml(deal) }); } // Call in routes/deals.ts after transitionDeal(): await sendDealFundedEmail(deal.sellerEmail, updated) .catch(err => req.log.error({ err }, "Email failed")); // Always catch — email failure ≠ deal failure
Cost: Resend free tier = 3,000 emails/month. Paid = $20/month for 50k emails. Negligible vs SafeSend revenue.

💬 Twilio (SMS)

New dependency. Simple to add alongside email calls
# Install npm install twilio // sms.ts — new file import twilio from "twilio"; const client = twilio( process.env.TWILIO_ACCOUNT_SID, process.env.TWILIO_AUTH_TOKEN ); export async function sendSms( to: string, body: string ): Promise<void> { if (!to || !process.env.TWILIO_PHONE_NUMBER) return; await client.messages.create({ from: process.env.TWILIO_PHONE_NUMBER, to: formatNzNumber(to), body }); } // Usage in deals.ts: await sendSms(deal.sellerPhone, `SafeSend: Payment received for "${deal.title}". Ship now and enter your tracking number: safesend.nz/deals/${deal.id}` ).catch(() => {}); // silent fail
Cost NZ: ~$0.08/SMS via Twilio. A deal generating $39 fee sends ~6 SMS = $0.48. 1.2% of fee revenue.
